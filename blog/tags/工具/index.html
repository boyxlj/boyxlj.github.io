git的基本使用2

2023年1月18日 · 阅读需 4 分钟
Miraitowa
越努力，越幸运

git基础配置​

git config --global use.name="昵称"
git config --global use.email='邮箱地址'

git基本使用​

  • 查看当前仓库状态 
git status
  • 初始化仓库 
git init
  • 未跟踪 → 暂存
git add <文件名>    # 将文件添加至暂存区
git add .    # 将所有未追踪文件添加至暂存区
  • 提交至版本库
git commit -m "注释"    # 将已添加暂存的文件添加至仓库中
git commit -a -m "注释"    # 提交所有已修改的文件到仓库中(未追踪除外)

常用命令​

  • 移动文件 
git mv from to    # 文件重命名(from=>原文件名；to=>新文件名)
  • 删除文件 
git rm <文件名>
git rm <文件名> -f    # 强制删除文件

分支​

  • 查看所有分支 
git branch
  • 创建分支 
git branch <分支名>
  • 切换分支 
git checkout <分支名>
git switch <分支名>
  • 创建并切换分支 
git checkout -b <分支名>
git switch -c <分支名>
  • 删除本地分支 
git branch -d <分支名>
  • 合并分支 
git merge <分支名>
  • 提交分支至远程仓库
git push <远程地址> <本地分支名>:<远程分支名>
  • 删除远程仓库分支
git push <远程地址> :<远程分支名>

远程仓库​

  • 将本地库上传至远程库
git remote add origin https://github.com/boyxlj/git-demo.git

git branch -M main    # 修改分支名为main

git push -u origin mian    # 推送至远程仓库上

远程仓库常用命令​

git remote    #查看关联的所有远程库

git remote add <远程库别名> <远程地址>    # 关联远程库

git remote remove <远程库别名>    # 删除远程库

git push -u <远程库别名> <分支名>    # 向远程库推送代码，并和当前分支关联
git push -u <远程库别名> <本地分支名>:<远程分支名>

git clone <远程地址>    # 从远程库下载代码

git pull <远程地址>    # 从服务器上拉取代码并自动合并 


*** 注意：推送代码之前，一定要先从远程库中拉取最新的代码

标签​

  • 查看所有标签
git tag
  • 为当前提交记录添加标签
git tag <版本号>
  • 为指定提交记录添加标签
git tag <版本号> <提交id>
  • 将标签推送至远程
git push <远程库别名> <版本号>    # 推送指定标签至远程仓库
git push <远程库别名> --tags    # 推送所有标签至远程仓库
  • 删除本地标签
git tag -d <版本号>
  • 删除远程标签
git push <远程库别名> --delete <版本号>

.gitngore​

git忽略配置文件，项目根目录创建.gitngore文件

  • 示例
.DS_Store
node_modules
/dist
.history

local env files

.env.local .env.*.local

Log files

npm-debug.log yarn-debug.log yarn-error.log pnpm-debug.log

Editor directories and files

.idea .vscode .suo .ntvs* .njsproj .sln

git的基本使用1

2022年1月16日 · 阅读需 4 分钟
Miraitowa
越努力，越幸运

1.区域​

  • 工作区
  • 暂存区
  • 版本库

2.对象​

  • git对象
  • 树对象
  • 提交对象

3.常用命令​

命令含义
git config --global user.name "nihao"(首次必须设置)
git config --global user.email 自己的邮箱(首次必须设置)
git config --list查看当前的配置项
git --version当前git版本
clear清空
echo "nihao"echo "nihao" > test.txt(控制台输出/创建文件夹)
ll当前目录下的子文件平铺控制台
git init初始化本地仓库

4.高层命令​

命令含义
git init初始化仓库
git status查看文件的状态
git diff查看哪些文件修改还未暂存
git diff --staged查看哪些文件修改以及被暂存了，还未提交
git log查看提交的历史记录
git log --online查看提交的历史记录（精简）
git add ./将修改添加到暂存区（未跟踪时进行跟踪）
git commit进入git编辑器
git commit -a跳过暂存区
git commit -a -m 注释跳过暂存区并提交到版本库
git commit -m 注释将暂存区并提交到版本库
git reset --hard 版本号穿梭/(回退到上一个版本)
git revert 版本号穿梭/回退(撤销当前版本提交，重新生成一个提交记录)

5.分支指令:​

命令含义
git branch 分支名创建分支名
git branch -v查看分支
git checkout 分支名切换分支
git checkout -b 分支名创建并切换分支
git switch -c master切换分支
git merge 分支名把指定的分支合并到当前分支
git branch -d 本地分支名删除本地分支
git push oirgin 远程分支名:本地分支名将分支推送至远程仓库
git push oirgin :本地分支名移除远程分支

6.远程仓库的操作​

命令含义
git remote -v查看当前所有远程地址别名
git remote add 别名 远程地址另起别名
git push -u 别名 分支推送本地分支上的内容到回远程仓库 (-u添加后会追踪记录,再次推送或拉取时只需要 git push)
git clone 远程地址将远程仓库的内容克隆到本地
git pull 远程地址别名 远程分支名将远程仓库对于分支最新内容拉下来后与当前本地分支直接合并
git tag -a v-1.1.1 -m "注释"生成版本
git push origin --tags提交版本至远程仓库

完整命令​

git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin (仓库地址)
git push -u origin main

git remote add origin (仓库地址)
git branch -M main
git push -u origin main
